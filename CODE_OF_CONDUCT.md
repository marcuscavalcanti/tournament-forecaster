# Code Of Conduct

## Our Standard

Project spaces must support technical disagreement without harassment. Expected behavior includes using welcoming language, respecting different viewpoints and experiences, accepting constructive criticism, focusing on what is best for the project, and showing empathy toward other participants.

Unacceptable behavior includes sexualized language or attention, personal or political attacks, insults, threats, sustained disruption, public or private harassment, publishing another person's private information, or conduct that a reasonable maintainer considers inappropriate in a professional setting.

## Scope

This policy applies in repository issues, pull requests, reviews, discussions, and project-related public representation.

## Enforcement

Report conduct incidents privately through the repository maintainer's GitHub profile or, when the report also concerns a vulnerability or sensitive disclosure, GitHub Private Vulnerability Reporting. Maintainers will protect reporter privacy, investigate context, and apply a proportionate response: correction, warning, temporary restriction, or permanent removal.

Maintainers who do not follow this policy may be removed from moderation decisions by the other project maintainers. Enforcement decisions may be reconsidered when material new evidence is provided.
